# Security policy

## Supported version

Security fixes are currently made on the latest PicFlip source version.

## Reporting a vulnerability

Please do not publish an unpatched vulnerability in a public issue. Use
GitHub's private vulnerability reporting feature for this repository. If that
feature is not enabled, open a minimal issue asking the maintainer for a
private contact method without including exploit details or private files.

PicFlip processes media entirely on the user's computer. Please never attach
private documents or personal media to a public security report. A synthetic
test file and clear reproduction steps are preferred.

## Scope

Reports concerning PicFlip's own TypeScript or Rust code are in scope.
Vulnerabilities in FFmpeg, x264, LAME, PDF.js, pdf-lib, Tauri, or another
dependency should also be reported to the relevant upstream project.
